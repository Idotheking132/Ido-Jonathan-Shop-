## רכיבי המערכת

### 1. Frontend (HTML/CSS/JS)

```
public/
├── index.html    → עמוד התחברות
├── shop.html     → עמוד החנות (דורש אימות)
├── admin.html    → פאנל ניהול (דורש רול Admin)
└── style.css     → עיצוב מלא ורספונסיבי
```

**תכונות:**
- עיצוב מודרני עם גרדיאנטים
- תמיכה בעברית (RTL)
- רספונסיבי (מובייל + דסקטופ)
- מודלים לאישור קנייה
- טפסים דינמיים

---

### 2. Backend (Node.js + Express)

**server.js** - שרת ראשי:

```javascript
Routes:
├── GET  /                    → עמוד התחברות
├── GET  /shop                → עמוד חנות (דורש auth)
├── GET  /admin               → פאנל ניהול (דורש admin)
│
├── GET  /auth/discord        → התחלת OAuth2
├── GET  /auth/callback       → קבלת תשובה מ-Discord
├── GET  /auth/logout         → יציאה
│
├── GET  /api/user            → פרטי משתמש מחובר
├── GET  /api/products        → רשימת מוצרים (עם הנחות VIP)
├── POST /api/purchase        → ביצוע קנייה
│
└── Admin API:
    ├── GET    /api/admin/products      → כל המוצרים
    ├── POST   /api/admin/products      → הוספת מוצר
    ├── PUT    /api/admin/products/:id  → עדכון מוצר
    ├── DELETE /api/admin/products/:id  → מחיקת מוצר
    └── GET    /api/admin/purchases     → היסטוריית קניות
```

**Middleware:**
- `requireAuth` - בדיקת אימות
- `requireAdmin` - בדיקת רול Admin
- `express-session` - ניהול סשנים

---

### 3. Discord Bot (Discord.js)

**bot.js** - בוט Discord:

```javascript
Functions:
├── isUserInGuild(userId)           → בדיקה אם משתמש בשרת
├── getUserRoles(userId)            → קבלת רולים של משתמש
└── createTicket(...)               → יצירת ערוץ טיקט
    ├── יצירת ערוץ בקטגוריה
    ├── הגדרת הרשאות (משתמש + מנהלים)
    └── שליחת הודעת פרטים
```

**Intents נדרשים:**
- `Guilds` - גישה לשרת
- `GuildMembers` - גישה לחברים ורולים

## זרימת אימות (OAuth2)

```
1. משתמש לוחץ "התחבר עם Discord"
   ↓
2. הפניה ל-Discord Authorization URL
   https://discord.com/api/oauth2/authorize?
   client_id=...&redirect_uri=...&response_type=code&scope=identify+guilds
   ↓
3. משתמש מאשר הרשאות בדיסקורד
   ↓
4. Discord מפנה חזרה עם קוד:
   http://localhost:3000/auth/callback?code=ABC123...
   ↓
5. שרת מחליף קוד ב-Access Token:
   POST https://discord.com/api/oauth2/token
   ↓
6. שרת מקבל פרטי משתמש:
   GET https://discord.com/api/users/@me
   ↓
7. בוט בודק רולים בשרת:
   getUserRoles(userId)
   ↓
8. שמירת משתמש ב-Session
   ↓
9. הפניה לחנות או שגיאה
```

## זרימת קנייה

```
1. משתמש בוחר מוצר ולוחץ "קנה עכשיו"
   ↓
2. פתיחת מודל אישור
   - בחירת כמות
   - חישוב מחיר (כולל הנחת VIP)
   ↓
3. לחיצה על "אישור קנייה"
   ↓
4. POST /api/purchase
   ↓
5. בדיקות בשרת:
   ├─ קולדאון (7 ימים מקנייה אחרונה)
   ├─ מלאי מספיק?
   ├─ כמות מותרת (max_purchase)?
   └─ חישוב מחיר סופי (עם הנחת VIP)
   ↓
6. עדכון מסד נתונים:
   ├─ הפחתת מלאי
   ├─ רישום קנייה
   └─ הגדרת קולדאון
   ↓
7. יצירת טיקט בדיסקורד:
   ├─ יצירת ערוץ חדש
   ├─ הגדרת הרשאות
   └─ שליחת הודעה עם פרטים
   ↓
8. החזרת תשובה למשתמש:
   "הקנייה בוצעה בהצלחה! נפתח לך טיקט בדיסקורד"
```

## אבטחה

### 1. אימות
- OAuth2 עם Discord
- Session-based authentication
- Secret key לסשנים

### 2. הרשאות
- בדיקת רולים בכל בקשה
- Middleware לבדיקת Admin
- בדיקה שהמשתמש בשרת

### 3. Validation
- בדיקת קלט משתמש
- בדיקת מלאי
- בדיקת קולדאון
- הגבלת כמות קנייה

### 4. Rate Limiting
- קולדאון 7 ימים בין קניות
- מניעת spam
